## Output Directory Structure

By default, output is organized under `out_dir/XPRESS_PROCESSING/`:

```
out_dir/
├── XPRESS_PROCESSING/
│   ├── config/              # Configuration files
│   ├── logs/               # Pipeline logs
│   ├── barcodes/           # Barcode-related files
│   ├── expression/         # Expression matrices and H5AD
│   ├── stats/              # Statistics and plots
│   ├── intermediate/       # Temporary files
│   └── *.bam              # Final BAM files
└── outs/                   # Final outputs for users
    ├── <project>_auto_plate_report.html  # Automatic report
    ├── <project>_manual_report.html      # Manual report
    ├── expression/                       # H5AD and MEX matrices moved from XPRESS_PROCESSING/expression/
    ├── stats/                            # QC tables, read stats, saturation tables, and plots
    ├── bam/                              # Final UB-corrected BAM and index
    └── config/                           # run_config.yaml and expected barcode table
```

`XPRESS_PROCESSING/` is the working directory used while the pipeline is running.
MfsFlow supports resuming from intermediate stages if a run stops before completion.
After a successful report/export step, customer-facing deliverables are moved to
`outs/`; completed runs should be consumed from `outs/`, not resumed from the moved
working outputs.

## Configuration Files

Located in `XPRESS_PROCESSING/config/`:

### run_config.yaml
Complete run configuration in YAML format. Contains all parameters used for the run.

**Purpose**: Reproducibility and provenance tracking.

**Format**: YAML

Example:
```yaml
project: "Sample01"
sample:
  sample_type: auto
  sample_id: 1
reference:
  STAR_index: /path/to/reference/star
  GTF_file: /path/to/reference/genes/genes.gtf
out_dir: /path/to/output
num_threads: 20
# ... (complete configuration)
```

### expect_id_barcode.tsv
Expected barcode table with well IDs and barcode sequences.

**Purpose**: Barcode matching.

**Format**: TSV with columns:
- `well_id`: Well identifier (e.g., "A1", "B2")
- `internal_barcode`: Internal barcode sequence
- `umi_barcode`: UMI barcode sequence (if applicable)

Example:
```
well_id	internal_barcode	umi_barcode
A1	ACGTACGTACGTACGTACGT	
A2	TGCATGCATGCATGCATGCA	
```

## Log Files

Located in `XPRESS_PROCESSING/logs/`:

### pipeline.log
Main pipeline log with timestamps.

**Purpose**: Debugging and monitoring.

**Format**: Text file with timestamp-prefixed messages.

Example:
```
2026-05-25 10:30:15 Start analysis for Sample01.
2026-05-25 10:30:16 Directories created.
2026-05-25 10:30:20 Fastq processed.
2026-05-25 10:30:25 Barcode files created.
...
```

### pipeline_timing.tsv
Timing information for each pipeline stage.

**Purpose**: Performance analysis.

**Format**: TSV with columns:
- `stage`: Pipeline stage name
- `start_time`: Stage start time
- `end_time`: Stage end time
- `duration_seconds`: Stage duration in seconds

Example:
```
stage	start_time	end_time	duration_seconds
Filtering	2026-05-25 10:30:20	2026-05-25 10:35:45	325.2
Mapping	2026-05-25 10:35:46	2026-05-25 11:15:22	2376.8
Counting	2026-05-25 11:15:23	2026-05-25 11:22:10	407.5
Summarising	2026-05-25 11:22:11	2026-05-25 11:25:33	202.1
```

### Stage-specific logs
Individual log files for each pipeline stage (if available).

## Barcode Files

Located in `XPRESS_PROCESSING/barcodes/`:

### kept_barcodes.tsv
List of barcodes that passed filtering.

**Purpose**: Quality control of barcode selection.

**Format**: TSV with columns:
- `barcode`: Barcode sequence
- `count`: Number of reads with this barcode
- `well_id`: Assigned well ID (if matched)

### barcode_binned.tsv
Barcode binning results (Hamming distance collapsing).

**Purpose**: Shows barcode correction results.

**Format**: TSV with columns:
- `original_barcode`: Original barcode sequence
- `binned_barcode`: Binned barcode sequence
- `hamming_distance`: Hamming distance between original and binned

### barcode_discovery_report.tsv
Barcode discovery results (only when no barcode mode is specified, i.e., discovery mode).

**Purpose**: Documents barcode inference.

**Format**: TSV with columns:
- `observed_barcode`: Barcode observed in reads
- `matched_barcode`: Matched barcode from bundled lists
- `match_type`: "exact" or "hamming-1"
- `count`: Number of reads with this barcode

## Expression Files

Located in `XPRESS_PROCESSING/expression/`:

The same H5AD and MEX deliverables are moved to `outs/expression/` after report
generation.

### MEX Format (Matrix Exchange)
Standard single-cell expression matrices. Each matrix type is written as a
separate MEX directory:

```text
XPRESS_PROCESSING/expression/
├── <project>.exon.umi/
├── <project>.intron.umi/
├── <project>.inex.umi/
├── <project>.exon.read/
├── <project>.intron.read/
└── <project>.inex.read/
```

The most commonly used matrix is `<project>.inex.umi`, which combines exon and
intron UMI counts. Read-based matrices use read counts instead of UMI counts.

#### matrix.mtx.gz
Sparse expression matrix in Matrix Market format.

**Purpose**: Gene expression counts.

**Format**: Sparse matrix (genes × cells) in Matrix Market format, gzipped.

#### features.tsv.gz
Gene annotations.

**Purpose**: Gene metadata.

**Format**: TSV with columns:
- `gene_id`: Gene ID (e.g., ENSG000001234)
- `gene_name`: Gene name (e.g., ACTB)
- `feature_type`: Feature type (e.g., "Gene Expression")

#### barcodes.tsv.gz
Cell barcode annotations.

**Purpose**: Cell metadata.

**Format**: TSV with columns:
- `barcode`: Cell barcode sequence
- `well_id`: Well ID (if available)

#### Reading MEX in Python

The MEX matrix is stored as genes x cells. Most downstream tools expect cells x genes,
so transpose the matrix after loading it manually.

```python
import pandas as pd
import scipy.io
import anndata as ad

matrix_dir = "XPRESS_PROCESSING/expression/Sample01.inex.umi"

X = scipy.io.mmread(f"{matrix_dir}/matrix.mtx.gz").tocsr().T
features = pd.read_csv(
    f"{matrix_dir}/features.tsv.gz",
    sep="\t",
    header=None,
    names=["gene_id", "gene_name", "feature_type"],
)
barcodes = pd.read_csv(
    f"{matrix_dir}/barcodes.tsv.gz",
    sep="\t",
    header=None,
    names=["barcode"],
)

adata = ad.AnnData(X=X)
adata.obs_names = barcodes["barcode"].astype(str).values
adata.var_names = features["gene_id"].astype(str).values
adata.var["gene_name"] = features["gene_name"].astype(str).values
```

#### Reading MEX with Scanpy

```python
import scanpy as sc

adata = sc.read_10x_mtx(
    "XPRESS_PROCESSING/expression/Sample01.inex.umi",
    var_names="gene_symbols",
    cache=False,
)
```

#### Reading MEX in R

```r
library(Matrix)

matrix_dir <- "XPRESS_PROCESSING/expression"
counts <- readMM(file.path(matrix_dir, "matrix.mtx.gz"))
features <- read.delim(file.path(matrix_dir, "features.tsv.gz"), header = FALSE)
barcodes <- read.delim(file.path(matrix_dir, "barcodes.tsv.gz"), header = FALSE)

rownames(counts) <- features$V2
colnames(counts) <- barcodes$V1
```

### <project>.h5ad
AnnData file for Python/Scanpy.

**Purpose**: Seamless integration with Scanpy.

**Format**: HDF5-based AnnData format.

**Contents**:
- `X`: Expression matrix (cells × genes)
- `obs`: Cell metadata (n_genes_by_counts, total_counts, etc.)
- `var`: Gene metadata (gene_name, etc.)
- `uns`: Unstructured metadata (QC metrics, etc.)
- `layers`: Additional matrices, such as `exon_umi`, `intron_umi`, `inex_umi`, `exon_read`, `intron_read`, and `inex_read`

#### Reading H5AD in Python

Use either `anndata` or `scanpy`:

```python
import anndata as ad
import scanpy as sc

adata = ad.read_h5ad("XPRESS_PROCESSING/expression/Sample01.h5ad")
# or:
adata = sc.read_h5ad("Sample01.h5ad")

# Main expression matrix
X = adata.X

# Optional layers, if present
exon_umi = adata.layers.get("exon_umi")
inex_read = adata.layers.get("inex_read")
```

Example Scanpy workflow:
```python
import scanpy as sc

adata = sc.read_h5ad("XPRESS_PROCESSING/expression/Sample01.h5ad")

# Basic QC
sc.pp.calculate_qc_metrics(adata)
sc.pl.violin(adata, ['n_genes_by_counts', 'total_counts'], jitter=0.4, multi_panel=True)

# Filtering
sc.pp.filter_cells(adata, min_genes=200)
sc.pp.filter_genes(adata, min_cells=3)

# Normalization
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)

# Dimensionality reduction
sc.pp.highly_variable_genes(adata, min_mean=0.0125, max_mean=3, min_disp=0.5)
adata = adata[:, adata.var.highly_variable]
sc.tl.pca(adata, svd_solver='arpack')
sc.pp.neighbors(adata, n_neighbors=10, n_pca=40)
sc.tl.umap(adata)
sc.pl.umap(adata, color=['total_counts', 'n_genes_by_counts'])
```

#### Reading H5AD in R

```r
library(reticulate)

ad <- import("anndata")
adata <- ad$read_h5ad("XPRESS_PROCESSING/expression/Sample01.h5ad")
counts <- adata$X
obs <- adata$obs
var <- adata$var
```

## Statistics Files

Located in `XPRESS_PROCESSING/stats/`:

Key statistics tables and plots are moved to `outs/stats/`.

### qc_stats.tsv
Per-cell QC statistics.

**Purpose**: Quality control metrics for each cell.

**Format**: TSV with columns:
- `cell_barcode`: Cell barcode
- `total_reads`: Total reads for this cell
- `mapped_reads`: Mapped reads
- `mapping_ratio`: Fraction of mapped reads
- `num_genes`: Number of genes detected
- `num_umis`: Number of UMIs detected
- `umi_fraction`: Fraction of reads with valid UMI
- `exon_intron_ratio`: Fraction of reads mapping to exons/introns

### saturation_<downsample_level>.tsv
Saturation analysis results at different downsampling levels.

**Purpose**: Gene detection saturation analysis.

**Format**: TSV with columns:
- `downsample_level`: Number of reads downsampled to
- `num_cells`: Number of cells detected
- `num_genes_median`: Median number of genes per cell
- `num_umis_median`: Median number of UMIs per cell

### gene_body_coverage.tsv
RSeQC-like gene body coverage profile.

**Purpose**: 5' to 3' coverage bias analysis. The pipeline selects the longest
transcript per gene from the GTF, splits the transcript body into 100 bins in
5' to 3' orientation, and projects primary aligned reads onto those bins.
UMI and Internal reads are tracked separately, with an additional combined
All-read profile in the text/PDF outputs.

**Format**: TSV with columns:
- `Percentile`: Gene body percentile (1-100)
- `UMI_Coverage`, `Internal_Coverage`, `All_Coverage`: Bin-normalized coverage
- `UMI_Frac`, `Internal_Frac`, `All_Frac`: Fractional coverage profiles
- `UMI_MaxNorm`, `Internal_MaxNorm`, `All_MaxNorm`: Max-normalized profiles used for plotting

### q30_stats.tsv
Base quality statistics.

**Purpose**: Sequencing quality metrics.

**Format**: TSV with columns:
- `read_type`: Read type (R1, R2)
- `q30_fraction`: Fraction of bases with Q30+
- `mean_quality`: Mean base quality

### Plots (PNG/SVG)
Generated plots (if `make_stats: yes`):

- `read_distribution.png`: Read distribution pie chart
- `saturation_curve.png`: Saturation curve plot
- `gene_body_coverage.png`: Gene body coverage plot
- `qc_metrics.png`: QC metrics violin plots

## Intermediate Files

Located in `XPRESS_PROCESSING/intermediate/`:

**Warning**: These are temporary files and can be deleted after successful completion.

### Split FASTQ Files
Split FASTQ files for parallel processing.

**Format**: Gzipped FASTQ.

Example:
```
Sample01_R1_split_001.fq.gz
Sample01_R1_split_002.fq.gz
...
```

### Intermediate BAM Files
Intermediate BAM files from STAR alignment.

**Format**: BAM (Binary Alignment Map).

Example:
```
Sample01_Aligned.out.bam
Sample01_Aligned.out.sorted.bam
Sample01_Aligned.out.sorted.bam.bai
```

### Corrected BAM Files
Barcode-corrected BAM files.

**Format**: BAM with corrected barcodes in CB tag.

## Final BAM Files

Located in `XPRESS_PROCESSING/`:

The final UB-corrected BAM and its index are moved to `outs/bam/`.

### <project>.filtered.Aligned.GeneTagged.UBcorrected.sorted.bam
Final UB-corrected sorted BAM file.

**Purpose**: Final alignment file with corrected barcodes and gene tags.

**Format**: Sorted BAM with the following tags:
- `CB`: Corrected cell barcode
- `UB`: Corrected UMI barcode
- `GN`: Gene name
- `GI`: Gene ID

**Usage** (Python/pysam):
```python
import pysam

bam = pysam.AlignmentFile("Sample01.filtered.Aligned.GeneTagged.UBcorrected.sorted.bam", "rb")

for read in bam:
    cell_barcode = read.get_tag("CB")
    umi_barcode = read.get_tag("UB")
    gene_name = read.get_tag("GN")
    # ... process read
```

## Final Reports

Located in `outs/`:

### auto_report.html
Automatic HTML report.

**Purpose**: Quick quality assessment.

**Contents**:
- Sample overview statistics
- Read distribution plots
- Mapping statistics
- QC metrics

### manual_report.html
Manual HTML report (more detailed).

**Purpose**: In-depth analysis.

**Contents**:
- Sample overview statistics
- UMI-based vs read-based statistics
- Per-well QC distribution
- Saturation curves
- Gene body coverage plots
- Detailed per-well statistics table

**Usage**: Open in any modern web browser.

## File Formats

### FASTQ
Standard FASTQ format with quality scores.

### BAM/SAM
Binary/Text Alignment Map format.

**Tools**:
- View BAM: `samtools view`
- Sort BAM: `samtools sort`
- Index BAM: `samtools index`
- Stats: `samtools flagstat`

### MEX (Matrix Market)
Sparse matrix format for single-cell data.

**Tools**:
- Python: `scipy.io.mmread`, `scanpy.read_mtx`
- R: `Matrix::readMM`, `Seurat::ReadMtx`

### H5AD
AnnData HDF5 format.

**Tools**:
- Python: `anndata.read_h5ad`, `scanpy.read_h5ad`
- R: `reticulate` (call Python from R)

### TSV/CSV
Tab-separated / Comma-separated values.

**Tools**:
- View: `less`, `head`, `tail`
- Process: `awk`, `grep`, Python/pandas, R
